# DeepLearningShogi 学習プロセスのやさしい説明

この文書は、将棋AIの学習を初めて触る人に向けて、全体の流れを短く説明したものです。

## 1. 全体像

学習は次の4段階です。

1. 対局データ（CSA）を集める
2. 学習用データ形式（hcpe / hcpe3）に変換する
3. ニューラルネットを学習する
4. 学習したモデルを実行形式（ONNXなど）に変換して対局に使う

## 2. なぜ変換が必要か

- CSAは「棋譜を保存する形式」です。
- そのままでは学習が遅いため、`csa_to_hcpe.py` などで学習向け形式に変換します。
- 変換後データを使うと、`train.py` が効率よく学習できます。

## 3. 最低限の実行フロー

WSL2上（Docker + uv環境）で次を行います。

1. 仮想環境を作成して依存ライブラリを入れる  
   - `uv venv .venv`  
   - `source .venv/bin/activate`
2. CSAを学習形式に変換する  
   - `dlshogi/utils/csa_to_hcpe.py`  
   - 必要なら `dlshogi/utils/csa_to_hcpe3.py`
3. 学習を実行する  
   - `python dlshogi/train.py ...`
4. 学習済みモデルを変換する  
   - `python dlshogi/convert_model_to_onnx.py ...`
5. Windows側で将棋GUIやfloodgate運用に使う

詳しいコマンドは次を参照してください。  
`docs/floodgate_csa_to_training_and_match.md`

## 4. 学習時に見るべきポイント

- **損失（loss）が下がるか**  
  下がらない場合は、学習率・バッチサイズ・データ量を見直します。
- **検証データで悪化していないか**  
  訓練だけ良くて検証が悪いと過学習の可能性があります。
- **データ件数が十分か**  
  極端に少ないと強くなりません。

## 5. よくあるつまずき

- 依存関係の不足（`torch`, `cshogi` など）
- ファイルパス違い（WSL2とWindowsで場所がずれる）
- GPU設定ミス（Dockerの `--gpus all` 付け忘れ）

## 6. 学生実験の進め方（推奨）

1. まず小規模データで1回最後まで通す（成功体験を作る）
2. 次にデータ量を増やして再学習する
3. 学習条件を1つだけ変えて比較する（同時に多く変えない）
4. 結果をログで残し、再現できるようにする

## 7. まとめ

重要なのは「まず最後まで動かすこと」です。  
最初から強いモデルを狙うより、**変換 → 学習 → 変換 → 対局** を一周できることが最初の目標です。
