# モデル構造（ResNet）と活性化関数の基礎

この文書は、将棋AIで使われるニューラルネットの見方を、初学者向けに整理したものです。

## 1. モデルは何をしているか

将棋AIのモデルは、局面特徴を入力して次を出力します。

- Policy head: 次の手の確率
- Value head: 局面の勝ちやすさ

中身（backbone）は、画像処理で有名な**ResNet系**がよく使われます。

## 2. ResNet とは

ResNetの要点は「**スキップ接続（残差接続）**」です。

通常の層は

- `x -> 層 -> y`

ですが、ResNetでは

- `x -> 層 -> f(x)`
- `x + f(x) -> y`

という形にします。  
これで深いネットでも学習しやすくなります。

## 3. なぜスキップ接続が効くのか

- 深いモデルは勾配が弱くなりやすい
- 何層も通ると情報が劣化しやすい
- 足し算で元の情報を直接流せるため、学習が安定しやすい

つまり、ResNetは「深くしても壊れにくい設計」です。

## 4. 活性化関数とは

活性化関数は、層の出力に非線形性を入れる関数です。  
これがないとモデルは単純な線形変換の重ね合わせになり、表現力が不足します。

### ReLU

- 定義: `max(0, x)`
- 長所: 速い、実装が簡単、広く使われる
- 短所: 負の領域で勾配0になりやすい（死んだReLU）

### Leaky ReLU

- 負の領域にも小さな傾きを残す
- ReLUの死にやすさを緩和

### GELU / SiLU (Swish)

- Transformerなどでよく使われる
- ReLUより滑らかで、タスクによっては性能向上
- ただし計算コストは少し高め

## 5. 将棋AIでの使い分けの考え方

- まずは実績の多い設定（例: ResNet + ReLU系）で開始
- 計算資源に余裕があれば活性化を比較実験
- 変更は1つずつ行い、勝率・loss・速度を同時に見る

## 6. モデルの出力ヘッド

ResNet本体の後ろに、通常2つのヘッドを付けます。

- Policy head: 多クラス分類に近い
- Value head: 回帰または分類（勝敗）

1つのbackboneを共有して、2つの目的を同時に学習する設計です。

## 7. 実験時のチェック項目

1. 学習が進んでいるか（lossが下がるか）
2. Valueだけ改善してPolicyが悪化していないか
3. 推論速度が実運用に耐えるか
4. GPUメモリ使用量が増えすぎていないか

## 8. よくあるつまずき

- モデルを深くしすぎて学習が不安定
- 活性化変更で速度が大きく低下
- パラメータ数を増やしたのに強くならない

強さは「モデルだけ」で決まらず、データ品質・探索設定・学習設定も重要です。

## 9. まとめ

押さえるべき要点は次の3つです。

1. ResNetはスキップ接続で深い学習を安定化する
2. 活性化関数は表現力と学習安定性に影響する
3. 最終性能はモデル単体ではなく、探索と学習条件の組み合わせで決まる
