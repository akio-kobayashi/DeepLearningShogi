# Policy・Value・MCTS のやさしい説明

この文書は、将棋AI学習でよく出る3つの考え方を、数式をほとんど使わずに説明します。

## 1. まず全体像

将棋AIは、ざっくり言うと次の分担で動きます。

- **Policy（方策）**: 「次にどの手を指すとよさそうか」を予測
- **Value（価値）**: 「この局面がどれくらい勝てそうか」を予測
- **MCTS（探索）**: PolicyとValueを使って、実際に読む手順を賢く絞る

つまり、  
「候補手を出す（Policy）→ 局面評価する（Value）→ 探索で最終決定（MCTS）」  
という流れです。

## 2. Policy とは

Policyは、局面を入力すると「各合法手の確率」を出すモデルです。

- 出力例:  
  - 7g7f: 0.32  
  - 2g2f: 0.21  
  - 8h2b+: 0.01

役割は「強そうな手の優先順位づけ」です。  
全手を同じ重みで読むと遅すぎるため、まずPolicyで有望手を上位にします。

## 3. Value とは

Valueは、局面から「勝率に近い値」を出します。

- 例: 0.85 ならかなり有利、0.50 なら互角、0.10 なら不利

役割は「この局面の見込み」を早く見積もることです。  
長い読みを全部やり切らなくても、途中局面の良し悪しを判断できます。

## 4. MCTS とは

MCTS（Monte Carlo Tree Search）は、木構造で探索する手法です。

1. ルート局面から候補手を展開する
2. 「まだ十分に調べていないが有望そうな手」を選ぶ
3. 進んだ先の局面をValueで評価する
4. 結果を親ノードに戻して統計を更新する
5. これを何度も繰り返し、訪問回数の多い手を選ぶ

ポイントは、**読むべき場所に計算を集中する**ことです。  
全分岐を同じ深さまで読むのではなく、有望な枝へ計算資源を寄せます。

## 5. 3者の関係（直感）

- Policy: 「まずどこを読むか」を案内する地図
- Value: 「その地点は良いか悪いか」を判定する採点係
- MCTS: 地図と採点を使って、限られた時間で最善手を探す探索エンジン

どれか1つだけでは弱く、組み合わせることで実戦的な強さになります。

## 6. 学習では何をしているか

学習時は主に次を最適化します。

- **Policy loss**: 正解に近い手確率を出せるようにする
- **Value loss**: 勝敗（または評価値）を当てられるようにする
- **合計損失**: PolicyとValueの誤差を重み付きで足し合わせる

実装ではログに `policy loss`, `value loss`, `total loss` のように出ることが多いです。

## 7. よくある誤解

- 「Policyだけ高精度なら強い」  
  → 探索と組み合わせないと実戦では不十分です。
- 「Valueだけ当たれば強い」  
  → 候補手の優先度が弱いと探索効率が悪くなります。
- 「探索回数を増やせば必ず強い」  
  → モデル品質が低いと、たくさん読んでも誤った枝に時間を使います。

## 8. 実験で比較するときのコツ

1. 1回に1条件だけ変える（例: 探索回数だけ変更）
2. 同じ対局条件で比較する（持ち時間・定跡有無を固定）
3. 勝率だけでなく、探索速度や損失推移も記録する

## 9. まとめ

将棋AIの要点は次の1行です。  
**Policyで候補を絞り、Valueで見込みを評価し、MCTSで限られた時間内に最善手を選ぶ。**

この3つの役割を分けて理解すると、学習ログや実験結果が読みやすくなります。
