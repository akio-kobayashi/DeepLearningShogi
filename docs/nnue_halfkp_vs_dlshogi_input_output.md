# NNUE（HalfKP）とdlshogiの入出力の違い

この文書は、将棋AIでよく混同される次の違いを整理します。

1. 入力: NNUEのHalfKP と dlshogiの入力特徴
2. 出力: NNUEの評価値 と dlshogiのPolicy/Value

## 1. 先に要点

- NNUE（HalfKP）は、**評価関数向けの特徴表現**を使い、出力は主に**局面評価値（1本）**
- dlshogiは、局面テンソルから**Policy（手の分布）**と**Value（勝ちやすさ）**を同時に出力

つまり、NNUEは「評価器中心」、dlshogiは「方策+価値の2ヘッド構成」です。

## 2. 入力の違い

## 2.1 NNUEのHalfKP

HalfKPは、王位置と駒配置の関係を使う特徴設計です。

- `K`: 自玉（または視点側の王）位置
- `P`: 盤上の駒特徴
- 王位置と駒特徴の組み合わせを高次元特徴として扱う

実装上の狙い:

- 探索中の局面更新で差分計算しやすい
- CPUで高速に評価しやすい
- 「1手進んだときに変わる部分だけ」更新しやすい

## 2.2 dlshogiの入力

dlshogiは、盤面をチャネル化したテンソル入力を使う方式が中心です。

- 駒種・先後・持ち駒・手番などを複数チャネルで表現
- CNN/ResNet系で空間的な特徴を抽出
- 局面全体のパターンを学習で獲得

実装上の狙い:

- 特徴設計を手作業で固定しすぎず、ネットに学習させる
- Policy/Valueの同時学習につなげやすい

## 3. 出力の違い

## 3.1 NNUEの出力

NNUEの出力は、基本的に**局面評価値**です。

- 例: centipawn相当のスカラー評価
- 探索側（alpha-beta等）がこの評価値を使って手を比較
- 「どの手が有望か」は探索で決まる比重が大きい

## 3.2 dlshogiの出力

dlshogiは通常2つの出力を持ちます。

- **Policy head**: 合法手に対する確率分布
- **Value head**: 勝率や勝敗見込みに対応する値

探索（MCTS）では、

- Policyで展開優先度を決める
- Valueで葉局面評価を行う

という使い方になります。

## 4. 対応表

| 観点 | NNUE（HalfKP） | dlshogi |
|---|---|---|
| 主な入力 | HalfKP特徴（王位置×駒特徴） | 盤面テンソル（複数チャネル） |
| 特徴設計 | 手設計要素が強い | 学習で表現獲得する比重が高い |
| 主な出力 | 評価値（スカラー） | Policy分布 + Value |
| 探索との関係 | 評価器として探索に供給 | 探索方針と評価を同時に供給 |
| 実行特性 | CPU高速化と差分更新が強み | GPU活用で表現力を引き出しやすい |

## 5. 初学者が混同しやすい点

1. Valueと評価値は似ているが用途が同一とは限らない  
   - NNUEの評価値は探索評価器として直接使う設計が中心  
   - dlshogiのValueはPolicy/MCTSと組み合わせて使う
2. PolicyがないNNUEは弱い、とは言えない  
   - NNUE系は探索と組み合わせて強い
3. 入力表現の違いは速度と実装方針に直結する  
   - HalfKPは更新効率  
   - dlshogi入力は表現学習の柔軟性

## 6. まとめ

- **入力の違い**: HalfKPは評価高速化に寄せた特徴、dlshogiは学習表現に寄せたテンソル
- **出力の違い**: NNUEは評価値中心、dlshogiはPolicy/Valueの2出力
- **設計思想の違い**: NNUEは探索評価器最適化、dlshogiは探索と学習出力の統合最適化
